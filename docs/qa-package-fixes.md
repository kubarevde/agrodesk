# QA checklist — пакет фиксов AgroDesk (календарь / смены / ввод / настройки)

## Блок 1 — Числовой ввод
- [ ] ТМЦ → новая позиция → ёмкость: ввод `100` без ведущего `0`
- [ ] Приход / списание ТМЦ: количество пустое до ввода
- [ ] Затраты / отгрузки / ставка сотрудника: то же
- [ ] Редактирование существующей записи с `0` — ноль остаётся видимым

## Блок 2–6 — Агрокалендарь
- [ ] Создать задачу с 2+ полями → сохранить → в карточке видны все поля
- [ ] Редактировать задачу: сменить поля / комментарий / приспособление
- [ ] Приспособление доступно без выбора техники (весь справочник)
- [ ] Клик по дню → «Добавить задачу на эту дату» → дата уже заполнена
- [ ] Комментарий виден в day view и (укороченно) в месячной ячейке

## Блок 7–8 — Смены
- [ ] Открыть смену при уже открытой → текст «У сотрудника уже есть открытая смена…» (не axios 409)
- [ ] Добавить смену 14.07 18:00 → 15.07 01:30 → успех
- [ ] Добавить пересекающуюся смену → конкретная ошибка пересечения
- [ ] end раньше start (тот же день) → понятная ошибка

## Регрессии (обязательно после деплоя)
- [ ] У сотрудника есть открытая смена с 16.07 → добавить закрытую 01.07–02.07 → **успех**
- [ ] Открытая с 16.07 08:00 → добавить 16.07 07:00–09:00 → **блок** с текстом о пересечении
- [ ] Агрокалендарь: создать задачу (1 поле и несколько) → без 500
- [ ] Агрокалендарь: отредактировать задачу → поля/комментарий сохраняются
- [ ] На VPS после деплоя: `alembic upgrade head` (или restart API-контейнера) → revision `015_agro_plan_fields`

## Блок 9 — Настройки mobile
- [ ] 360–414px: вкладки горизонтально скроллятся, не наезжают на заголовок/текст
