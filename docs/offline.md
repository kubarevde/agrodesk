# Офлайн-режим АгроДеск

Кратко: приложение открывается без сети **после** первого онлайн-визита и установки.  
Офлайн-запись сейчас — **только смены**. Остальное — онлайн или просмотр из кэша.

---

## Для пользователя

### Подготовка (обязательно онлайн)
1. Откройте сайт **с интернетом** и войдите.
2. Дождитесь индикатора **«Онлайн»** в шапке.
3. Нажмите **«Установить приложение»** (Android Chrome) или Safari → **«На экран Домой»**.
4. Один раз откройте приложение **из иконки** ещё онлайн — так закрепляется service worker.

Готовность: в шапке «Онлайн», меню открывается, раздел «Рабочее время» загружается.

### Работа без интернета
1. Запускайте **из иконки**, не из сохранённой вкладки браузера.
2. В шапке — **«Офлайн»** и баннер.
3. В **«Рабочее время»** открывайте/закрывайте смены → «Сохранено офлайн».
4. Жёлтое число в шапке = очередь синхронизации.

### После появления сети
1. Откройте приложение (или дождитесь «Онлайн»).
2. Очередь уходит сама.
3. Если остались ошибки (красный счётчик) — нажмите **обновление (Повторить)** в шапке.
4. Убедитесь, что счётчик исчез.

### Вкладка ≠ PWA

| Способ | Офлайн |
|--------|--------|
| Сохранённая вкладка | Часто **не** открывается / нестабильно |
| «На главный экран» / Установить | Shell из кэша после прогрева — **рекомендуется** |

### Если очередь зависла
1. Включите интернет.
2. Нажмите «Повторить» в шапке.
3. Перезапустите приложение из иконки.
4. Если не помогло — обратитесь к администратору (не удаляйте данные вручную).

## Границы offline support

| Модуль | Read офлайн | Write офлайн | Примечание |
|--------|-------------|--------------|------------|
| App shell / меню | Да* | — | *после онлайн-визита + SW |
| Auth bootstrap | Да* | — | кэш профиля; 401 сбрасывает сессию |
| Смены | Да* | **Да** | Dexie + syncQueue |
| Справочники (объекты, типы работ…) | Да* | Нет | если уже кэшировались |
| Техника / поля / склад | Частично* | Нет | локальный кэш при наличии |
| Затраты / отгрузки | Нет / UI-заглушка | Нет | online only |
| Дашборд | Нет / last stats | Нет | online only |
| Отчёты | Нет | Нет | online only |
| Финансы KPI | Нет | Нет | через dashboard API |

\* Зависит от предварительного кэширования при работе онлайн.

## Для разработки

### Как тестировать
```bash
npm run build && npm run preview
# DevTools → Application → Service Workers (должен быть activated)
# Network → Offline → reload (сессия не должна уходить на /login)
npm test -- src/features/auth/authOffline.test.ts
npm run test:e2e -- e2e/shifts.spec.ts
```

### Важно
- Service worker **только в production** (`build` / `preview` / деплой), не в `npm run dev`.
- Auth: `agrodesk_user_cache` + токен; офлайн не вызывает `/api/auth/me`.
- `flushSyncQueue` — mutex (startup + online + retry не гоняются параллельно).
- Retry вручную: `includeFailed: true` переводит failed → pending.
- Deploy/S3: `index.html`, `sw.js`, `manifest.webmanifest` — `no-cache` (см. `.github/workflows/deploy.yml`).
- Nginx: отдельные `location` для этих файлов с `Cache-Control: no-cache`.

### Типовые поломки после деплоя
1. `sw.js` / `index.html` с долгим cache → старый shell / вечный старый SW.
2. `VITE_BASE_PATH` без relative manifest → сломан install/start_url.
3. Auth снова дергает `/me` без fallback → logout офлайн.
4. Ожидание offline write вне смен — пока не реализовано.

### Manual QA checklist (релиз)
- [ ] Онлайн: логин, «Онлайн», SW installed/activated (prod).
- [ ] Установка PWA / Add to Home Screen.
- [ ] Открыть из иконки онлайн.
- [ ] Выключить сеть → открыть из иконки → **не** `/login`.
- [ ] Баннер офлайн виден.
- [ ] Открыть смену офлайн → очередь +1 → reload → запись на месте.
- [ ] Закрыть смену офлайн → очередь.
- [ ] Включить сеть → очередь очищается / Retry работает.
- [ ] Дашборд/Отчёты офлайн показывают честный online-only notice.
- [ ] Android Chrome + (по возможности) iOS Safari.

### iOS Safari
- «На экран Домой» — основной способ.
- Ограничения SW жёстче, чем у Chrome; всегда прогревайте онлайн после установки.
